Gestion de processus métiers

 

TP BPEL

 

 

Objectif

L’objectif du TP est de suivre un tutoriel pour déployer votre premier processus BPEL appelé « Hello World» qui reçoit un message (une chaine de caractères) du client et répond en renvoyant le même message. Il sera composé d’une séquence d’activités :

une activité Receive, suivi d’une activité Assign (qui va copier le message reçu en entrée dans le message de sortie) et d’une activité Reply. 

 

Les outils utilisés sont BPEL designer , Apache Orchestration Director Engine (ODE) sur Eclipse  pour simuler des processus BPEL.

 

1.     Suivez les instructions données en TP pour installer les modules nécessaires et pour configurer Eclipse :

http://manager.crio.dauphine.fr/sharing/dgrigori/TP-BPEL/

2.     Suivez les étapes du tutoriel ci-dessous pour déployer votre premier processus BPEL « Hello World».

 

Deploying a "Hello World" BPEL

Right click on the BPEL_HelloWorld project, select New->Others->BPEL 2.0->New BPEL Process File. Click Next. Fill in BPEL Process Name the string HelloWorld, and in the Namespace the string http://helloworld. Select the Template as Synchronous BPEL Process. Click Finish.

  1. The BPEL Editor will now open and show the created process. To add another actions or controls, you have to use the BPEL Palette. To show this Palette, right click on the empty space in the Editor and select Show Palette in Palette View.
  2. We want to design an application that response the client the same string that it send to the service provider. Therefore, we have to add an activity which assign the received input to the output. To do this, click on the Assign activity on the Actions folder of the Palette and drag it to the space between the receiveInput and replyOutput activities.

  1. Right click on the Assign activity. Select Show in properties.
  2. In the Properties tab, select Details. Click on the New button to define an assign operator.
  3. Assign from input->payload->input to output->payload->result. An Initializer popup dialog appear. Click Yes to initialize the output variable and save the process.

  1. The BPEL process is designed. Now we have to define the Service, Binding type and PortType address to run the HelloWorld service. Right click on the HelloWorldArtifacts.wsdl file, select Open With->WSDL Editor.
  2. Right click on the  Binding icon, select ShowPorperties. In the Properties view, set the name of the new binding as HelloWorldSOAPBinding and select PortType as HelloWorld.

  1. Click on Generate Binding Content... button. A Binding wilzard appears. Select the Protocol as SOAP and SOAP Binding Options as document literal. Click Finish.

  1. Click on the Service. Rename the service as HelloWorldService
  2. Click on the port below the service's name. Then, in the Properties view, change the port's name to HelloPort. Select the Binding as HelloWorldSOAPBinding. Set the address to the HelloWorld process's location on the local server, which is http://localhost:8080/ode/processes/HelloWorld and the Protocol is SOAP. (Attention : insert ode/processes in the string !)

  1. Save the file, you have the WSDL design like this.

  1. To let the Apache ODE know your declarations and designs. You have to add a descriptor. Right click on the BPEL_HelloWorld project, select New->Others->BPEL 2.0->Apache ODE Deployment Descriptor.
  2. Click Next. Verify the BPEL Project name as /BPEL_HelloWorld. Click Finish.
  3. Now you have the file deploy.xml in your project. Right click on this file. Select Open With->ODE Deployment Descriptor Editor.

  1. In the Inbound Interfaces table, select the Partner Link as client, Associate Port as HelloPort. Click on Related Service then other fields will be automatically filled. Save the "deploy.xml" file.
  2. Now you have finished your design and it's time to run the process. However, the ODE Server on the Eclipse could not recognize your BPEL process because it is designed after the server's set up. Therefore you have to restart your Eclipse.
  3. After the restart, right click on the Ode v1.x Server at localhost, select Add and Remove Projects.
  4. Select the BPEL_HelloWorld/HelloWorld in the Available projects box and click Add. Then, click Finish.
  5. Start the ODE server. If the deployment was successful, you will see the following output in the console tab.

  1. We will test the "Hello World" deployment with Web Services Explorer provided by Eclipse. Right click on the HelloWorldArtifacts.wsdl file. Select Web Services->Test with Web Services Explorer.
  2. Select the operation process. Then, type Hello ODE in the input textbox. If your deployment was success, you will get the same string in the SOAP response.

  1. Done

Exercices

1.     Modifiez le processus pour qu’il retourne "Hello "+Input.  Par exemple, si l’entrée est "John", le processus répond "Hello John".

2.     Définissez un processus BPEL qui a deux opérations: sayHello and sayGoodbye qui retourne "Hello "+Input et "GoodBye "+Input. (utilisez l’activité Pick)
